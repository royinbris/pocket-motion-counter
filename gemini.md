# Project: Pocket Motion Counter (주머니 운동 카운터)

## Goal (목표)
휴대폰을 사용자의 바지 앞주머니에 넣은 상태에서 모션 센서(가속도 및 자이로) 데이터를 사용하여 스쿼트 등의 운동 반복 횟수를 자동으로 측정하는 모바일 웹 앱을 개발합니다.

## Product Direction (제품 방향)
- 카메라 기반의 포즈 디텍션은 이번 구현 범위에서 제외합니다.
- 주머니에 휴대폰을 넣은 상태(Phone-in-Pocket)에서의 움직임 감지를 최우선으로 지원합니다.
- 첫 단계 운동은 스쿼트(Squat)에 한정하여 개발합니다.
- 실험 및 테스트 조건은 “우측 바지 앞주머니”로 고정합니다.
- 모션 신호를 가공하는 코어 엔진(`motion-core`)을 UI 프레임워크와 분리하여 차후 다른 웹앱에도 간편하게 탑재할 수 있도록 구조화합니다.

## Technical Rules (기술 규칙)
- 개발 언어는 기본적으로 **TypeScript**를 사용합니다.
- 코드는 테스트가 쉽도록 작고 단일한 책임을 갖는 모듈 단위로 작성합니다.
- 신호 처리(Signal Processing) 알고리즘과 카운팅 상태 머신은 UI 라이브러리와 완벽히 분리하여 순수 함수 및 순수 클래스로 설계합니다.
- 정적 배포(Static Hosting)가 원활하도록 별도의 서버 백엔드 없는 클라이언트 기반 구조로 개발합니다.
- 외부 라이브러리 의존성을 최소화하고, 브라우저 표준 API(`DeviceMotionEvent`)를 최우선으로 활용합니다.
- HTTPS 환경에서의 테스트 및 배포를 단순 릴리즈가 아닌 일상적인 개발 루프의 일부로 포함시킵니다.

## Workflow Rules (워크플로 규칙)
- 코드를 작성하기 전에 항상 구현 계획 아티팩트를 먼저 세웁니다.
- 개발은 점진적인 단계로 실행합니다: `데이터 수집 -> 신호 분석 -> 카운터 구현 -> 기기 검증`
- 각 단계가 마무리될 때마다 검증 노트를 작성합니다.
- 스쿼트 카운팅이 정확하고 안정적으로 동작할 때까지 다른 운동으로 확장하지 않습니다.
- 알고리즘의 고도화보다 데이터 수집의 정합성과 일관된 모바일 기기 디버깅 환경 마련을 우선합니다.

## Testing Rules (테스트 규칙)
- 데스크톱 에뮬레이션에만 의존하지 않고, 실제 모바일 기기에서 적극적으로 교차 검증합니다.
- 브라우저 보안 규격에 맞게 센서 권한 요청 흐름은 반드시 사용자의 명시적인 터치 이벤트(Interaction) 후에 트리거되도록 합니다.
- 실제 배포된 HTTPS URL을 활용하여 실기기에서 최종 동작 여부를 확인합니다.
- 디버깅과 알고리즘 튜닝을 용이하게 하기 위해, 유저 세션의 센서 가속도/자이로 raw 로그를 저장하고 텍스트로 내보낼 수 있는 기능을 포함합니다.
- 오탐지(False Positive)와 미탐지(False Negative)를 구분하여 정량적으로 추적하고 개선합니다.

## Architecture Rules (아키텍처 규칙)
- 애플리케이션(`apps/`)과 재사용 가능한 패키지(`packages/`)를 명확하게 분리하는 모노레포 구조를 유지합니다.
  - `packages/motion-types`: 센서 데이터 데이터 형식 및 카운터 정보 관련 공통 선언 패키지.
  - `packages/motion-core`: 신호 가공, LPF(Low-Pass Filter), 크기(Magnitude) 계산, 피크 추출 및 FSM 상태 관리.
  - `apps/sensor-lab`: 운동 중 발생하는 실시간 센서 변화 그래프 및 로우 로그 데이터 수집을 담당하는 R&D용 실험 앱.
  - `apps/counter-demo`: 일반 사용자 시나리오(목표 회수 입력, 카운트 뷰어, 시작/정지 제어, 비프음/진동 알림)를 시뮬레이션하는 데모 앱.
- 코어 엔진은 향후 다양한 앱에 임베딩할 수 있도록 `start()`, `stop()`, `onCount()`, `onComplete()` 등의 이벤트 기반 외부 인터페이스(API)를 설계합니다.

## Deployment Rules (배포 규칙)
- 코드의 모든 변경 사항은 GitHub 저장소를 기준으로 추적합니다.
- 1차 배포 환경으로 **Vercel**을 사용하여, 모노레포 내의 각 프론트엔드 앱이 HTTPS 도메인으로 즉시 자동 배포되도록 합니다.
- `main` 브랜치는 항상 동작 및 빌드가 완료되어 즉시 실기기에서 다운로드해 동작을 볼 수 있는 클린한 상태로 관리합니다.

## Non-Goals (비목표)
- 이번 페이즈에서는 카메라나 포즈 디텍션 인공지능 모델을 사용하지 않습니다.
- 규칙 기반(Rule-based) 및 물리 모션 데이터 기반 카운팅이 제대로 기능하는 한 AI 머신러닝 모형은 활용하지 않고 단순하고 명확한 알고리즘을 지향합니다.
- 다중 운동 지원이나 다중 주머니 위치 감지는 이번 범위에서 고려하지 않습니다.
