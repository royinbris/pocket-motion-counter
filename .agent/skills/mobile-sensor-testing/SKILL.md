# Skill: Mobile Sensor Testing (모바일 실기기 센서 검증)

이 스킬은 데스크톱 환경에서는 시뮬레이션하기 어려운 센서 데이터를 실제 모바일 브라우저 환경에서 재현성 있게 테스트하고 검증하는 구체적인 실기기 평가 루프에 대해 기술합니다.

## 사용 시점
- 모바일 실기기 상에서 센서 동작을 직접 테스트하고 정상 작동 유무를 수동 검증할 때.
- Vercel에 배포된 Preview 링크를 모바일 폰으로 최초 접근하거나 권한 문제를 디버깅할 때.
- 가상의 노이즈 시나리오(걷기, 뛰기, 제자리 앉기 등)를 모바일로 직접 수행하면서 카운트 정밀성을 평가할 때.

## 수행 지침

### 1. 실기기 테스트 준비 단계
- **연결 방법**:
  - **로컬 HTTPS의 경우**: 터미널에 표시된 Local IP 주소(예: `https://192.168.x.x:5173`)를 QR 코드로 변환해 스마트폰 카메라로 스캔하여 접속합니다. (동일 LAN 망 Wi-Fi 접속 필수)
  - **Vercel 호스팅의 경우**: 자동 배포 완료 후 제공되는 HTTPS URL(Preview 또는 Production)로 직접 접속합니다.
- **센서 부착 상태 고정**:
  - 테스트 대상자는 우측 바지 앞주머니에 스마트폰을 꼽습니다. 
  - 기본 가이드: **상단(카메라 부분)이 위를 향하고 화면이 몸쪽**을 바라보는 상태를 기준으로 삼아 일관성을 높입니다.

### 2. 표준 테스트 스크립트 실행
테스트 진행 시 매번 아래 기록 시트를 기록하고, 발생한 False Positive와 False Negative 내역을 분류합니다.

| 테스트 ID | 수행 시나리오 | 목표 수행 수 | 인지된 카운트 | 판정 결과 | 특이 사항 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| S-01 | 정속 스쿼트 10회 | 10회 | | PASS/FAIL | |
| S-02 | 빠른 스쿼트 10회 | 10회 | | PASS/FAIL | 템포 영향도 평가 |
| N-01 | 제자리 걷기 (30초) | 0회 | | PASS/FAIL | 오탐율 검증 |
| N-02 | 일상적인 앉았다 일어서기 3회 | 0회 | | PASS/FAIL | 오탐율 검증 |

### 3. 디버깅 세션 로그 백업 절차
테스트 결과 정확도가 떨어질 경우(FAIL 판정 시):
1. `sensor-lab` 앱에서 동일 동작에 대한 데이터를 다시 기록하고 시작/종료합니다.
2. 수집된 JSON 로그를 로컬 드라이브로 저장(Export)합니다.
3. `/data/sample-logs/` 디렉터리에 `{오동작유형}-{날짜}.json` 파일명으로 저장하여 알고리즘 개선을 위한 모의 데이터(Mock Data)로 편입시킵니다.
