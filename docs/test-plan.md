# Test Plan: Pocket Motion Counter

모바일 환경에서의 실기기 테스트 및 센서 오탐지 방지 검증을 위한 테스트 계획서입니다.

## 1. 테스트 목적
- 모바일 기기가 바지 주머니에 들어간 실 사용 조건에서 운동(스쿼트)을 정확히 인식하는지 검증합니다.
- 단순 일상 동작(걷기, 일어서기, 앉아 있기)을 운동으로 오인하여 카운트가 올라가는 현상(False Positive)을 차단합니다.
- 정상적인 스쿼트 수행 시 센서 흔들림 등으로 인해 카운트가 누락되는 현상(False Negative)을 방지합니다.

## 2. 기본 테스트 조건
- **테스트 기기**: 스마트폰 (iOS Safari 및 Android Chrome 교차 테스트)
- **장착 위치**: 우측 바지 앞주머니 (Right-Front Pants Pocket)
- **방향**: 스마트폰 상단이 위 또는 아래를 향하든 상관없이 카운팅되도록 지향하되, 초기 튜닝 단계에서는 스마트폰 충전 단자가 아래를 향하게 고정하여 테스트합니다.

## 3. 테스트 시나리오 및 통과 기준

### 시나리오 A: 정상 스쿼트 수행 (카운팅 감도 테스트)
- **절차**:
  1. 주머니에 휴대폰을 넣고 카운팅 앱을 켭니다 (권한 승인 완료).
  2. 스쿼트를 정속으로 10회 수행합니다.
  3. 완료 후 표시된 카운트를 확인합니다.
- **통과 기준**: 카운트 개수가 정확히 `10`이어야 함 (오차 허용 범위 $\pm 0$회 목표, 최대 $\pm 1$회 허용).

### 시나리오 B: 일상 동작 수행 (오탐지 방지 테스트)
- **절차**:
  1. 주머니에 휴대폰을 넣은 상태에서 30초간 제자리 걷기(Walking)를 수행합니다.
  2. 3회 앉았다가 일어서기를 매우 천천히 일상 속도로 반복합니다.
- **통과 기준**: 운동 카운트가 `0`으로 유지되어야 함.

### 시나리오 C: 동작 정지 상태 (센서 드리프트 노이즈 테스트)
- **절차**:
  1. 기기를 책상 위에 평평하게 내려놓고 30초간 대기합니다.
  2. 기기를 들고 주머니에 넣은 채 가만히 서 있습니다.
- **통과 기준**: 누적 카운트가 전혀 발생하지 않아야 함 (`0`).

## 4. 데이터 로깅 및 튜닝 프로토콜
센서 신호 분석 및 알고리즘 디버깅을 위해 `sensor-lab` 앱에서 다음 데이터를 수집하여 분석합니다.
1. **Raw Log 구조**:
   - `timestamp` (ms)
   - `accelX`, `accelY`, `accelZ` (중력 포함 3축 가속도)
   - `gyroAlpha`, `gyroBeta`, `gyroGamma` (3축 자이로 회전 속도)
2. **세션 리포트**:
   - 실제 수행 운동 횟수(사용자 수동 기입)와 수집 데이터의 파형 비교 분석.
   - 엑셀 또는 데이터 가시화 도구를 통해 피크(Peak) 임계값의 마진(Margin)을 재설정하여 상태 머신 임계치 값(`THRESHOLD_DOWN`, `THRESHOLD_UP`) 튜닝.
