# Product Brief: Pocket Motion Counter

휴대폰 센서를 이용한 주머니 속 운동 카운팅 서비스 기획 요약입니다.

## 1. 배경 및 목적
기존의 운동 카운팅 모바일 앱은 주로 전면/후면 카메라를 활용한 Pose Estimation(자세 추정)에 의존합니다. 하지만 이 방식은 다음과 같은 명확한 단점이 있습니다.
- 휴대폰을 적절한 각도로 거치할 수 있는 공간이 필요함.
- 야외나 협소한 공간, 혹은 헬스장처럼 공용 공간에서는 카메라 설치가 부담스러움.
- 배터리 소모와 기기 발열이 매우 심함.

**Pocket Motion Counter**는 사용자의 바지 주머니(초기 스펙은 우측 앞주머니)에 휴대폰을 넣은 상태에서 오직 모션 센서(가속도, 자이로)만을 이용하여 운동 횟수를 인지함으로써, 거치의 번거로움을 해결하고 일상에 녹아드는 운동 카운팅 경험을 제공하고자 합니다.

## 2. Target Exercise & Condition (대상 운동 및 조건)
- **대상 운동**: 스쿼트 (Squat)
- **센서 위치**: 우측 바지 앞주머니 (Right-Front Pants Pocket)
- **기기 제약**: 화면이 꺼져 있거나 백그라운드 환경인 상태에서도 카운팅이 되도록 구현하는 것이 최종 지향점이나, 웹 브라우저의 보안 제약 상 초기 MVP 버전은 **화면이 켜진 포그라운드 상태의 모바일 웹 브라우저**에서 구동하는 것을 전제로 합니다.

## 3. 핵심 기능 요구사항 (MVP)
- **Sensor Lab 앱 (앱 1)**:
  - 기기 가속도(Linear Acceleration + Gravity) 및 자이로(Rotation Rate) 실시간 센서 스트림 수집 및 가시화.
  - 디버깅용 3축 원본 데이터 세션 녹화 및 파일(JSON) 내보내기.
  - 운동 메타데이터 기입 (예: "스쿼트 10회 실시").
- **Counter Demo 앱 (앱 2)**:
  - 목표 횟수를 설정하고 카운팅을 시작할 수 있는 대시보드 UI.
  - 스쿼트 상승/하강 국면을 유한 상태 머신(FSM)으로 모니터링하여 오탐지 없이 카운팅.
  - 스쿼트 1회 완료 시마다 "비프 소리" 및 "진동(안드로이드 지원 기기)" 피드백.
  - 목표 횟수 달성 시 완료 연출 브레이크다운 화면.
- **Motion Core 패키지 (엔진)**:
  - 노이즈를 제어하기 위한 이동 평균(Moving Average) 또는 지수 이동 평균(EMA) 로우패스 필터링.
  - 중력 가속도 성분을 제외한 선형 가속도의 크기(Magnitude) 및 피치/롤 각도 변화 분석.
  - 다양한 브라우저와 하이브리드 앱에 쉽게 포팅할 수 있는 이벤트 기반 API 구조 (`onCount`, `onStateChange`).

## 4. 비기능적 요구사항
- **반응성**: 센서 샘플링 레이트는 50Hz 이상을 지향하여 실시간 피드백 지연을 최소화합니다.
- **배포 편의성**: 정적 빌드 파일로 빌드되어 Vercel을 통해 HTTPS 환경에 즉시 정적 서빙이 가능해야 합니다.
